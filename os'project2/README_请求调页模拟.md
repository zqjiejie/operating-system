# 请求调页存储管理方式模拟

本项目实现操作系统课程项目中的“请求调页存储管理方式模拟”，不包含动态分区分配部分。

## 功能

- 作业共有 320 条指令，地址空间为 32 页。
- 每页存放 10 条指令。
- 为作业分配 4 个内存块。
- 初始时所有页均未调入内存。
- 指令访问序列按 PPT 中的局部性原则随机生成：
  - 50% 指令顺序执行；
  - 25% 指令均匀分布在前地址部分；
  - 25% 指令均匀分布在后地址部分。
- 支持 FIFO 和 LRU 两种页面置换算法。
- 每次访问输出页号、页内偏移、命中/缺页、内存块号、物理地址、置换页和当前内存块状态。
- 执行结束后统计缺页次数和缺页率。

## 运行方法

```bash
python request_paging_sim.py
```

指定 LRU 算法：

```bash
python request_paging_sim.py --algorithm LRU
```

指定随机种子，便于复现实验结果：

```bash
python request_paging_sim.py --algorithm FIFO --seed 2026
```

显示全部 320 条访问记录：

```bash
python request_paging_sim.py --algorithm FIFO --seed 2026 --verbose
```

## 输出说明

- 物理地址计算方式：`内存块号 * 页面大小 + 页内偏移`。
- 当前内存块显示 4 个内存块中装入的页号，`--` 表示空块。
- 缺页率计算方式：`缺页次数 / 访问指令总数`。
