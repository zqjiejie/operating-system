# 电梯调度系统

这是一个操作系统课程项目示例：20 层楼、5 部互联电梯，使用 Python 标准库 `threading` 和 `tkinter` 实现。

## 运行方式

在 VS Code 终端中进入项目目录：

```powershell
cd E:\桌面\Elevator
python elevator_dispatch.py
```

不需要安装第三方依赖。若系统存在多个 Python 版本，也可以尝试：

```powershell
py elevator_dispatch.py
```

## 已实现功能

- 20 层、5 部电梯，全部初始停在 1 层。
- 每部电梯独立线程运行，模拟电梯并发工作。
- 每部电梯内部有 1-20 数字键、开门、关门、报警/解除报警按钮。
- 每层楼门口有上行、下行按钮。
- 楼层按钮互联：同一层的上行/下行请求只维护一份共享状态，按下后所有电梯共同响应该请求。
- 每层每部电梯门口都有数码显示，显示该电梯当前楼层和运行方向。
- 电梯无上层或下层相关请求时保持原地不动。
- 报警后该电梯暂停运行，解除报警后继续参与调度。

## 调度算法

调度器维护两类请求：

- `car_requests`：某部电梯轿厢内部产生的目标楼层请求。
- `hall_requests`：楼层门口产生的共享外部请求，包含楼层和方向。

每部电梯线程循环执行：

1. 向调度器申请任务。
2. 如果当前楼层有本电梯内部请求或已分配的外部请求，则开门服务。
3. 如果有目标楼层，按当前方向继续移动；没有同方向请求时选择最近的目标。
4. 如果没有任何相关请求，则进入空闲状态并停留原地。

外部请求分配采用“唯一分配 + 最近优先 + 顺路优先”：

- 同一层同一方向的外部请求只分配给一部电梯执行，其他电梯不会同时出发。
- 空闲电梯按距离最近优先。
- 正在运行的电梯如果方向一致且顺路，优先级更高。
- 不顺路的运行电梯会增加惩罚分，减少反复掉头。
- 被分配电梯报警后，该外部请求会释放并重新分配给其他电梯。
